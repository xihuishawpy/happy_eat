# Domain Docs

This repository uses a single domain context.

## Before exploring

- Read the root `CONTEXT.md`.
- Read ADRs under `docs/adr/` that affect the work.
- If either source is absent, proceed without creating it preemptively.

## Vocabulary

Use the domain terms defined in `CONTEXT.md` in task titles, acceptance criteria, tests, and implementation notes. Avoid synonyms that the glossary explicitly rejects.

## Architecture decisions

If proposed work conflicts with an ADR, state the conflict explicitly instead of silently overriding the decision.
